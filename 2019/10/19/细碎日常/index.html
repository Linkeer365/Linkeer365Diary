为什么开启”细碎日常”

还是觉得github.io主页的”设备使用细节-细碎日常”主要应该记录一些技术方面的东西, 鸡毛蒜皮的小事应该转移到这个diary的post下面比较合适些, 这里主要记录的是”非技术层面”的一些细碎的日常, 可以是心情, 可以是思考, 也可以是一些好吃的好看的好玩的这样世俗而琐屑的东西, 因为从整体上看, 平凡的我们的一生就是在这样平淡而偶有波澜的日常里度过的.

注意事项

  1. 不要害怕重复, 觉得在blog和diary两者都有意义的日常是可以两边都写的, 就是复制粘贴一下的功夫
  2. 不要觉得”哎呀这有什么好记的, 太细碎了”, 因为这个”细碎日常”更偏向于你当下的心情和思想, 说到底就是乱糟糟的杂七杂八的东西, 我们在blog或者colorfullife里头的确要做一点筛选, 以免困于大量的”日常”中得不到关键信息, 但是这里的日常我们就是要事无巨细一一写下来, 不管再小也好,都代表了你自己的人生.
  3. 要敏锐, 要非常尊重自己当下的感觉; 抓住每个当下去记录自己的感受, 这个很重要, 很多灵感就是稍纵即逝的, 要把他们都记录下来,
  4. 要诚实, 不要在意道德层面上对自己的无端谴责, 就是觉得我这样的思想真是怠惰or肮脏什么的, 因为诚实很多时候就是必须要直面丑恶, 现在丑恶是自己的, 自己还有不原谅自己的理由吗?
  5. 每个月份更新下##小标题, 搜狗输入法输入ny(有教程自定义一下)
  6. 每天更新下###小标题, 搜狗输入法输入nyr(以后注意下, ny和nyr就是”年月”和”年月日”, 注释不能更新, 坑爹!)

  • (2019年10月24日11:40:11) “待办事项”这个功能在hexo上可以渲染出来, 具体表现为
  1. 未完成事项 -> - [ ] sth -> -w[w]wsth(其中w换成空格)
  2. 已完成事项 -> - [x] sth -> -w[x]wsth
  3. hexo已完成标记为x, 有的网站可能标记为v或者其他字母(有的可能干脆就不支持这个语法), 视情况而定

就markdown编辑器而言, 这个语法可能部分编辑器没有支持, 只要最终网页渲染没有问题就可以了
